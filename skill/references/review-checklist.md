# 자체 평가·수정 체크리스트 (Review Checklist)

렌더링된 PNG를 **Read 도구로 직접 열어 한 장씩 눈으로 확인**합니다. 텍스트 양은
주제마다 달라서, 코드가 정상이어도 렌더 결과에서만 보이는 문제가 생깁니다. 그래서
이 루프는 생략하지 않습니다.

## 매 카드 확인 항목

**레이아웃·여백**
- [ ] 본문 마지막 줄이 **푸터(출처·페이지 번호)와 겹치거나 너무 붙지** 않았는가?
      → 가장 흔한 문제. `.card` 하단 패딩(기본 100px)이 푸터 공간을 확보하는지 확인.
        본문이 길면 해당 카드 본문 `font-size`/`line-height`/`margin-top`을 줄인다.
- [ ] 헤드라인과 본문 사이 빈 공간이 과하게 벌어지지 않았는가? (짧은 본문 카드)
      → `.body { margin-top }`을 줄여 헤드라인 아래로 자연스럽게 붙인다.
- [ ] 카드 안 요소들이 좌우 패딩 안에 있고 잘리지 않았는가?

**타이포·가독성**
- [ ] 헤드라인이 어색한 위치에서 줄바꿈되지 않는가? (`<br>` 위치 조정 또는 폰트 미세 축소)
- [ ] 본문 텍스트가 박스/카드 밖으로 넘치지 않는가?
- [ ] 배경 대비 글자 가독성이 충분한가? (특히 다크 카드, 하이라이트 위 글자)
- [ ] 형광펜(`.hlb`)을 한 카드에 1~2회만 썼는가? (남발 시 강조 효과 상실)

**일관성**
- [ ] kicker 번호와 footer 페이지 번호가 순서대로 맞는가?
- [ ] footer의 total(`NN / total`)이 실제 전체 장수와 일치하는가?
      → `function footer(no, total=N)`의 N을 전체 장수로. 표지/마무리의 인라인 푸터도 확인.
- [ ] 출처 표기가 전 카드 동일한가?
- [ ] 색 테마가 전 카드에서 통일됐는가? (다크/라이트 카드 모두 같은 BRAND/ACCENT 계열)

**내용**
- [ ] 한 카드 = 한 아이디어인가? (과밀하면 분할)
- [ ] 원본에 충실한가? 과한 재해석·없는 사실이 들어가지 않았는가?
- [ ] 의견/시사점이 명령조가 아니라 배움·관찰 톤인가?
- [ ] 숫자·고유명사·인용이 원본과 일치하는가?

## 수정 흐름
1. 문제를 발견하면 `build.js`의 해당 카드/CSS를 고친다.
2. `node build.js && node render.js`로 재렌더.
3. 영향받은 카드만 다시 Read로 확인.
4. 전 카드가 깨끗할 때까지 반복한 뒤 전달.

## 자주 나오는 수정 레시피
- **푸터 충돌** → `.card` 하단 패딩 확보 + 긴 본문 카드 폰트 20px·line-height 1.5로.
- **여백 쏠림** → `.body { margin-top:auto }` 대신 고정값(예: 30px)으로 바꿔 흐름 자연화.
- **헤드라인 3줄 균형 깨짐** → 폰트 46→42/40px로 낮추고 `<br>` 재배치.
- **다크 카드 강조색이 테마와 따로 놂** → 라임/블루/골드 같은 예시 틴트를 BRAND/ACCENT를
  밝게 한 값으로 교체.
