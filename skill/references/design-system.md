# 디자인 시스템 (Design System)

카드뉴스의 **구조·타이포·여백은 고정**, **색상 테마만 주제에 맞춰 교체**합니다.
일관된 시리즈 느낌을 유지하면서도 주제별 정체성을 줄 수 있게 하는 것이 목적입니다.

## 1. 색상 테마 선택

매 주제마다 `scripts/build.js` 상단의 THEME 상수 7개를 채웁니다. 고를 때 원칙:

- **메인 컬러(BRAND)**: 주제의 브랜드/감성을 대표하는 색. 출처가 특정 기업·제품이면
  그 브랜드 컬러를 쓰면 즉시 "그 주제답게" 보입니다. (Shopify=그린, 금융=네이비,
  헬스=틸/그린, AI/테크=인디고·바이올렛, 환경=포레스트그린, 경고/리스크=딥레드 등)
- **BRAND_DK**: 메인을 어둡게 한 톤. 다크 카드(표지·마무리·스탯) 배경과 소제목에 사용.
- **INK**: 본문 글자색. 순수 검정 대신 메인 계열이 살짝 섞인 거의-검정(가독성 + 통일감).
- **SUB**: 보조 텍스트. 회색에 메인을 약간 섞어 톤을 맞춤.
- **PAPER**: 본문 카드 배경. 순백보다 **채도 낮은 밝은 톤**(종이 베이지, 미색, 연회색)이
  눈이 편하고 고급스럽습니다. 주제가 차가우면 쿨그레이, 따뜻하면 웜베이지.
- **ACCENT**: 메인과 **대비되는** 포인트색. 강조 박스·장식 웨이브·두 번째 개념 구분에 사용.
  메인이 그린이면 블루, 네이비면 골드/오렌지처럼 보색·준보색을 고르면 위계가 또렷합니다.
- **LINE**: 구분선/테두리. PAPER보다 살짝 어두운 같은 계열.

### 테마 예시 팔레트

| 주제 성격 | BRAND | BRAND_DK | PAPER | ACCENT |
|---|---|---|---|---|
| 테크/그린(Shopify류) | `#5E8E3E` | `#3D6B26` | `#F4F1E9` | `#1F6FEB` |
| AI/소프트웨어 | `#5B5BD6` | `#3A3A9E` | `#F2F1FA` | `#E8590C` |
| 금융/신뢰 | `#1F4E79` | `#14385A` | `#F0F3F7` | `#C9A227` |
| 헬스/웰빙 | `#1F8A70` | `#15604D` | `#EFF5F2` | `#E8730C` |
| 환경/지속가능 | `#3A7D44` | `#285731` | `#F1F3EC` | `#1F6FEB` |
| 리스크/경고 | `#B23A3A` | `#7E2727` | `#F5EFEC` | `#1F4E79` |

이 표는 출발점일 뿐입니다. 주제에 더 맞는 색이 있으면 자유롭게 정하세요. 대비·가독성만
지키면 됩니다(본문 INK는 PAPER 위에서 충분히 진하게).

## 2. 고정 레이아웃 규격

- 캔버스: **800×800px**, 렌더 scale **2×**(deviceScaleFactor) → 선명한 PNG.
- 카드 패딩: `64px 60px 100px`(하단은 푸터 공간 확보를 위해 넓게).
- 폰트: **Noto Sans KR** (400/500/700/900). 한글 본문에 안정적.

## 3. 컴포넌트 해부 (baseCSS 기준)

표준 본문 카드는 위에서 아래로 이 순서를 지킵니다:

1. **kicker** — `.chip`(번호, 메인색 배경 흰 글자) + 라벨(BRAND_DK). 카드 주제를 한마디로.
2. **headline** — 46px/900, 자간 -1px, 2줄 내외. 한 문장 핵심 메시지. 키워드는
   메인색 `<span>`으로 칠해 시선을 잡음.
3. **divider** — 54×5px 메인색 둥근 막대. 헤드라인과 본문을 분리.
4. **body** — 23px/1.62. 단락 `<p>`. 강조는 두 종류:
   - `.hl` : 메인색 굵게 (핵심어)
   - `.hlb`: 형광펜 하이라이트(메인색 28% 배경) — 한 줄에 1~2회만, 남발 금지.
5. **footer** — 절대배치(하단 고정). 왼쪽 출처, 오른쪽 `NN / total` 페이지 번호(메인색).

### 강조 톤 박스 (포인트)
중요한 결론 한 줄은 흰 배경 + 메인/ACCENT 좌측 보더 박스로 강조하면 효과적:
```html
<p style="background:#fff;border-left:5px solid ${BRAND};border-radius:0 12px 12px 0;
   padding:18px 20px;margin-top:18px;font-weight:700;color:${BRAND_DK};">결론 문장</p>
```

### 다크 카드 (표지·마무리·스탯·구조도)
배경을 `linear-gradient`(BRAND_DK→더 어두운 톤), 글자 흰색, 강조는 밝은 메인 틴트
(예: 라임 `#A7E07A`)와 ACCENT 틴트로. 표지·마무리에 SVG 웨이브 장식으로 수미상관.

## 4. 정보전달(PPT형) 카드를 섞어라
본문 카드만 이어지면 단조롭습니다. 전달에 도움이 되는 곳엔 도식 카드를 끼웁니다:
- **스탯 카드**: 큰 숫자 2~3개를 박스로 (규모·임팩트 강조).
- **구조도 카드**: 레이어/스택/관계를 박스와 화살표로.
- **2단 비교 카드**: 대비되는 두 개념을 좌우 카드로(메인 vs ACCENT 보더).
- **체크리스트/원칙 카드**: 항목을 색 보더 박스로 나열.

구체적 마크업은 `card-patterns.md` 참고.
